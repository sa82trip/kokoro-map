# 마인드맵 앱 개발 프로젝트

## 프로젝트 개요
이 프로젝트는 PRD 문서를 기반으로 한 마인드맵 앱 개발을 위한 작업 공간입니다. Agile 방식으로 스프린트 단위로 진행되며, 각 스프린트는 명확한 사용자 스토리와 태스크로 구성됩니다.

## 디렉토리 구조

```
mindmap-app/
├── README.md                          # 프로젝트 개요
├── SPRINT-1.md                      # Sprint 1 계획서
├── sprints/                          # 모든 Sprint 폴더
│   ├── sprint-1/                    # Sprint 1: 기본 마인드맵 생성
│   │   ├── user-story-1/            # 사용자 스토리 1
│   │   │   ├── README.md            # 스토리 개요
│   │   │   └── tasks/               # 상세 태스크 목록
│   │   ├── user-story-2/            # 사용자 스토리 2
│   │   ├── user-story-3/            # 사용자 스토리 3
│   │   └── user-story-4/            # 사용자 스토리 4
│   ├── sprint-2/                    # Sprint 2: 파일 관리
│   ├── sprint-3/                    # Sprint 3: 웹 버전
│   ├── sprint-4/                    # Sprint 4: 실시간 협업
│   ├── sprint-5/                    # Sprint 5: 템플릿
│   ├── sprint-6/                    # Sprint 6: 모바일 앱
│   ├── sprint-7/                    # Sprint 7: AI 기능
│   ├── sprint-8/                    # Sprint 8: 발표 모드
│   ├── sprint-9/                    # Sprint 9: 통합 기능
│   └── sprint-10/                   # Sprint 10: 엔터프라이즈
└── backlog.md                       # 전체 백로그 문서
```

## 작업 흐름

### 1. Sprint 계획 단계
1. `SPRINT-[N].md`에서 Sprint 목표 확인
2. 해당 스프린트 폴더로 이동
3. 각 `user-story-[N]/README.md`에서 스토리 상태 확인

### 2. 개발 단계
1. `user-story-[N]/README.md`의 태스크 목록 확인
2. `tasks/` 폴더에 있는 각 태스크 파일에서 상세 구현 사항 확인
3. 태스크 완료 후 파일 상태 업데이트

### 3. 검토 단계
1. 스토리 완료 후 리뷰 진행
2. `README.md`의 상태 업데이트
3. 다음 Sprint로 이동

## Sprint 개요

### Sprint 1: 기본 마인드맵 생성 및 편집 (4주)
- **목표**: 중심 노드 생성, 노드 추가/삭제, 기본 편집 기능
- **스토리**: 4개 (26 포인트)

### Sprint 2: 파일 관리 및 저장 시스템 (4주)
- **목표**: 저장/불러오기, 검색, 폴더 구조
- **스토리**: 4개

### Sprint 3: 웹 버전 출시 준비 (4주)
- **목표**: 반응형 디자인, 성능 최적화, UI/UX
- **스토리**: 4개

### Sprint 4: 실시간 동시 편집 (4주)
- **목표**: WebSocket 기반 협업 시스템
- **스토리**: 4개

### Sprint 5: 템플릿 시스템 (4주)
- **목표**: 템플릿 라이브러리, 내보내기 기능
- **스토리**: 3개

### Sprint 6: 모바일 앱 출시 (4주)
- **목표**: React Native 기반 모바일 앱
- **스토리**: 1개

### Sprint 7: AI 보조 기능 (4주)
- **목표**: 자동 아이디어 추천, 키워드 분석
- **스토리**: 3개

### Sprint 8: 발표 모드 (4주)
- **목표**: 발표 뷰, 애니메이션, 보조 기능
- **스토리**: 3개

### Sprint 9: 통합 기능 (4주)
- **목표**: Google Drive, Slack 등과의 연동
- **스토리**: 2개

### Sprint 10: 엔터프라이즈 기능 (4주)
- **목표**: 관리자 대시보드, 보안 기능
- **스토리**: 2개

## 작업 지침

### 상태 관리
- **To Do**: 작업 전
- **In Progress**: 개발 중
- **In Review**: 리뷰 대기
- **Done**: 완료

### 태스크 완체 체크리스트
- [ ] 기능 구현 완료
- [ ] 단위 테스트 통과
- [ ] 코드 리뷰 완료
- [ ] 문서 업데이트
- [ ] UI/UX 검수 완료

## 작업 시작 방법
1. 현재 진행 중인 Sprint 확인
2. 해당 스프린트 폴더로 이동
3. 우선순위가 높은 스토리 선택
4. 태스크 상세 구현 사항 확인
5. 작업 시작

**주의**: 각 태스크 완료 시 상태를 반드시 업데이트해 주세요.
