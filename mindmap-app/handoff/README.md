# Handoff 관리 문서

## 개요
이 문서는 개발 작업의 전체 진행 상황을 파악하고, 팀 간의 작업 이동(handoff)을 원활하게 관리하기 위한 문서입니다. 각 스프린트별 진행 상태와 완료율을 추적합니다.

## Handoff 작성 지침
1. **스프린트 완료 시**: 해당 스프린트 폴더에 `HANDOFF.md` 파일 생성
2. **내용 업데이트**: 작업 완료 시마다 `current-status.md` 파일 업데이트
3. **정기 리뷰**: 매주 금요일 Handoff 문서 리뷰

## 파일 구조
```
handoff/
├── README.md                    # Handoff 관리 가이드
├── current-status.md            # 현재 진행 상태 (실시간 업데이트)
├── sprint-1/
│   └── HANDOFF.md              # Sprint 1 완료 보고서
├── sprint-2/
│   └── HANDOFF.md              # Sprint 2 완료 보고서
└── ...
```
