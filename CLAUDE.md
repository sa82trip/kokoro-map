# Claude 작업 지침: 마인드맵 프로젝트

## 프로젝트 개요
이 프로젝트는 PRD 문서를 기반으로 한 마인드맵 앱 개발입니다. Agile 방식으로 10개의 스프린트로 나뉘어 진행됩니다.

## 진행 관리 체계

### 1. 작업 구조

#### 파일 계층 구조
```
mindmap-app/
├── README.md              # 프로젝트 개요
├── dashboard.md           # 전체 대시보드 (실시간 업데이트)
├── handoff.md             # 작업 인계 문서 (시작 전 확인, 종료 후 업데이트)
├── CHANGELOG.md           # 변경 이력 (작업 완료 시 업데이트)
├── SPRINT-[N].md          # Sprint 계획서
└── sprints/              # 작업 공간
    ├── sprint-[N]/       # 각 스프린트
    │   ├── user-story-[M]/
    │   │   ├── README.md # 스토리 개요
    │   │   └── tasks/    # 상세 태스크
    │   └── ...
```

### 2. 업데이트 규칙

#### 매일 업데이트 필수 항목

1. **dashboard.md**
   - 주간 진행률 업데이트
   - 팀 생산성 지표 갱신
   - 다일 목표 체크

#### Sprint 중 업데이트
1. **스프린트 계획서 (SPRINT-[N].md)**
   - 스토리 상태 변경
   - 작업자 할당
   - 완료일 업데이트

2. **사용자 스토리 (user-story-[M]/README.md)**
   - 스토리 상태 변경
   - 실제 소요 시간 기록

3. **태스크 파일 (tasks/[ID].md)**
   - 작업 진행 상태 업데이트
   - 작업자 및 기간 기록

### 3. 상태 관리 코드

#### 스토리 상태
- `[ ] To Do` - 작업 전
- `[x] In Progress` - 개발 중
- `[x] In Review` - 리뷰 대기
- `[x] Done` - 완료

#### 태스크 상태
- `[ ] 설계 완료` - 설계 단계 완료
- `[x] 개발 중` - 개발 진행 중
- `[ ] 코드 리뷰 대기` - PR 생성 대기
- `[ ] 테스트 완료` - 테스트 통과
- `[ ] 완료` - 모든 단계 완료

### 4. 작업 흐름

#### Sprint 시작 시
1. `dashboard.md`에 Sprint 목표 입력
2. `SPRINT-[N].md` 작성
3. `handoff.md` 업데이트

#### 일일 작업
1. 작업 시작 전 `handoff.md` 확인
2. 작업 시작 시 태스크 상태 "개발 중"으로 변경
3. 작업 완료 시 상태 업데이트

#### Sprint 완료 시
1. `handoff.md` 최신 상태로 업데이트
2. `CHANGELOG.md`에 변경 사항 기록
3. `dashboard.md` 전체 진행률 업데이트
4. 다음 Sprint 계획 수립

### 5. 진행 모니터링

#### 핵심 지표
- **스토리 완료율**: 목표 100%
- **포인트 완도율**: 목표 100%
- **테스트 커버리지**: 80% 이상
- **코드 리뷰**: 모든 PR 필수

#### 경고 기준
- 연속 3일 진행률 0% → 리스크 등록
- 스토리 지연 5일 이상 → 우선순위 재조정
- 테스트 실패 3회 이상 → 품질 검토

### 6. 주요 리스크 관리

#### 리스크 등록 방법
1. `handoff.md`의 알려진 이슈 항목에 추가
2. 영향도와 긴급도 평가
3. 담당자 지정
4. 해결 계획 수립

#### 차단 요소(Blockers) 처리
1. 즉시 `handoff.md`에 등록
2. 데일리 스탠딩에서 논의
3. 24시간 내 해결책 마련

### 7. 문서화 표준

#### 파일 작성 규칙
- 파일명: 소문자와 하이픈 사용
- 제목: #을 사용한 H1 형식
- 상태: 체크박스 형식으로 관리
- 날짜: YYYY-MM-DD 형식 사용

#### 버전 관리
- 주요 변경 시 버전 업 (v1.0 → v1.1)
- 변경 내용 기록 필수
- 백업 파일은 별도 폴더 관리
