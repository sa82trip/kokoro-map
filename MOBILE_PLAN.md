# 모바일 대응 전략

## 현재 상태 분석

### 모바일 대응 현황
- ✅ 기본 반응형 레이아웃 (768px 미디어 쿼리)
- ❌ 터치 이벤트 미구현
- ❌ 모바일 터치 UI 제어기 없음
- ❌ 핀치 줌/아웃 지원 없음
- ❌ 모바일 최적화된 툴바/설정 UI
- ❌ 키보드 가림 현상 처리 없음
- ❌ 모바일 성능 최적화 부족

### 주요 문제점
1. **툴바가 너무 작은 버튼** - 모바터치하기 어려움
2. **드래그가 마우스에만 최적화** - 터치 드래그 지원 없음
3. **텍스트 편집 UI** - 모바일 키보드 처리 미흡
4. **설정 패널** - 모바일 친화적 UI 아님
5. **연결선 렌더링 성능** - 복잡한 마인드맵에서 느림

## 모바일 대응 플랜

### Phase 1: 기반 구조 (2-3일)
1. **터치 이벤트 핸들러 구현**
   - `useTouch` 훅 개발
   - 터치 드래그 지원
   - 스와이프/핀치 인식
   - 더블 탭/롱 프레스 지원

2. **반응형 레이아웃 강화**
   - 375px (iPhone SE), 414px (iPhone 12+), 768px (iPad) 기준 미디어 쿼리
   - 플렉서블 레이아웃으로 변경
   - 상대 단위 (rem, em) 사용
   - 터치 타겟 크기 보장 (48px 이상)

3. **모바일 터피커 UI**
   - 가상 패드 컴포넌트
   - 노드 선택 메뉴
   - 툴바 메뉴 → 하단 도크바로 변경

### Phase 2: 상호작용 개선 (3-4일)
1. **드래그/스크롤 최적화**
   - 터치 스무딩
   - 패닝 가속도 조절
   - 경계 처리 개선
   - 모멘텀 스크롤

2. **인풃/편집 기능**
   - 모바일 키보드 가림 처리
   - 텍스트 선택 메뉴 연동
   - 외부 키보드 지원
   - 자동 정렬 개선

3. **제스처 시스템**
   - 핀치 줌/아웃
   - 두 손가락 드래그로 패닝
   - 한 손가락 드래그로 노드 이동
   - 스와이프로 스토리 변경

### Phase 3: 성능 최적화 (2-3일)
1. **렌더링 최적화**
   - 가상화 연결선 렌더링
   - 노드 렌더링 디버깅
   - GPU 가속 활용
   - 지연 로딩

2. **저장/로드 최적화**
   - IndexedDB 지원
   - 점진적 로딩
   - 백그라운드 저장
   - 동기화 최적화

3. **네트워크 최적화**
   - 오프라인 모드
   - 점진적 동기화
   - 에러 재시도
   - 데이터 압축

### Phase 4: 고급 기능 (2-3일)
1. **PWA 기능**
   - 앱 스크린샷
   - 푸시 알림
   - 홈 화면 설치
   - 오프라인 지원

2. **접근성**
   - 스크린 리더 지원
   - 대비 모드
   - 글자 크기 조절
   - 음성 명령

3. **제스처 커스터마이징**
   - 제스매 설정 UI
   - 프로필 저장/로드
   - 단축 제스처
   - 접근성 제스처

## 기술 스택

### 라이브러리
- **Hammer.js** - 제스처 인식 라이브러리
- **react-use** - 터치/모바일 유틸리티 훅
- **mobile-friendly** - 모바일 UI 컴포넌트
- **react-native-web** - 모바일 최적화 (선택 사항)

### CSS
- `touch-action: manipulation`
- `user-select: none`
- `-webkit-tap-highlight-color: transparent`
- `overscroll-behavior: contain`

### 아키텍처
- 모듈화된 터치 시스템
- 플랫폼별 렌더링 옵션
- 성능 모니터링
- 접근성 레이어

### 중요도 보통
1. 오프라인 지원
2. 접근성
3. 고급 제스처
4. 커스터마이징

### 로드맵
- **Sprint 5**: 기반 구구조 (MVP)

## 성능 지표

### 목표 성능
- **FCP**: < 1.5s
- **LCP**: < 2.5s
- **FID**: < 100ms
- **CLS**: < 0.1
- **인터랙션**: < 150ms

### 모바일 특화 지표
- **터치 레이턴시**: < 100ms
- **드래그 FPS**: 60
- **줌 레이턴시**: < 200ms
- **메모리 사용량**: < 100MB

## 리스크 관리

### 주요 리스크
1. 성능 저하 - 가상화 및 최적화로 해결
2. 제스처 충돌 - 계층화된 이벤트 처리
3. 크로스 브라우저 - 테스트 커버리지 확보
4. 오프라인 동기화 - 충돌 처리 로직

- 모바일 전용 CDN
- 그래디언트 퍼포먼스 테스트
- 사용자 피드백 루프
- A/B 테스트
