# 마인드맵 앱 백로그 (Sprint Backlog)

## 개요
본 문서는 마인드맵 프로그램 PRD를 기반으로 한 스프린트 단위 스토리와 태스크 목록입니다.

---

## Phase 1: MVP 기능 구현 (3개월)

### Sprint 1: 기본 마인드맵 생성 및 편집

#### 스토리 1: 마인드맵 기본 구조 생성
**사용자 스토리**: 사용자는 중심 노드를 생성하고 마인드맵의 기본 구조를 만들 수 있어야 한다.

**태스크**:
- [ ] 중심 노드 컴포넌트 개발
- [ ] 중심 노드 텍스트 편집 기능 구현
- [ ] 기본 레이아웃 시스템 구축
- [ ] 새로운 마인드맵 생성 버튼 구현
- [ ] 마인드맵 제목 설정 기능
- [ ] 기본적인 상태 저장/로드 기능

#### 스토리 2: 노드 추가 및 삭제 기능
**사용자 스토리**: 사용자는 마인드맵에 새로운 노드를 추가하고 기존 노드를 삭제할 수 있어야 한다.

**태스크**:
- [ ] 노드 추가 버튼 UI 개발
- [ ] 키보드 단축키로 노드 추가 (Enter, Tab)
- [ ] 노드 삭제 기능 (Delete 키, 컨텍스트 메뉴)
- [ ] 노드 중복 선택 가능성 구현
- [ ] 삭제 확인 다이얼로그 구현
- [ ] 노드 위치 자동 배치 알고리즘

#### 스토리 3: 노드 편집 기능
**사용자 스토리**: 사용자는 노드의 텍스트를 편집하고 기본 스타일을 변경할 수 있어야 한다.

**태스크**:
- [ ] 노드 텍스트 직접 편집
- [ ] 텍스트 크기 조절 슬라이더
- [ ] 텍스트 색상 변경 기능
- [ ] 텍스트 굵기/이탤릭 체인지
- [ ] 노드 박스 크기 자동 조절
- [ ] 텍스트 입력시 자동 저장 기능

#### 스토리 4: 시각적 기본 기능
**사용자 스토리**: 사용자는 마인드맵의 시각적 요소를 기본적으로 변경할 수 있어야 한다.

**태스크**:
- [ ] 기본 색상 팔레트 구현 (50가지)
- [ ] 노드 배경색 변경 기능
- [ ] 연결선 스타일 선택 (직선, 곡선)
- [ ] 연결선 색상 변경
- [ ] 노드 간 거리 조절 기능
- [ ] 전체 레이아웃 재조정 기능
